# Phase 3: LLM-Generated Archetype Agents

Phase 3 replaces the hand-tuned parameter tables from Phase 1 with parameters
generated by an LLM (Claude Sonnet). The goal is to test whether LLM-generated
behavioral parameters can replicate (or diverge from) the trust dynamics
observed in Phase 1, where all 8 archetypes used researcher-defined parameter
tables.

## Architecture

```
phase3/
  personality_specs/     # Stage 1: Qualitative + quantitative archetype specs
    oracle.md
    sentinel.md
    firestorm.md
    wall.md
    phantom.md
    predator.md
    mirror.md
    judge.md
  generate_params.py     # Stage 2: Anthropic API script to generate params
  generated_params/      # Output: LLM-generated parameter JSON files
    (empty until generate_params.py is run)
  dealer.py              # Stage 3: Game integrity layer
  llm_agent.py           # Stage 4: LLM agent classes (LLMAgent, LLMPredator, LLMJudge)
  run_phase3.py          # Stage 5: Simulation runner
  __init__.py
  README.md              # This file
```

## Execution Order

### Step 1: Set API Key

```bash
export ANTHROPIC_API_KEY="sk-ant-..."
```

### Step 2: Generate Parameters

```bash
python3 phase3/generate_params.py
```

This reads the personality specs, constructs structured prompts with the
parameter template (keys and types only — no Phase 1 values), calls the
Anthropic API (model: `claude-sonnet-4-20250514`), validates the output, and
saves to `phase3/generated_params/`.

The script will **error-exit** if `ANTHROPIC_API_KEY` is not set. There is no
offline fallback — parameters must be generated by the LLM.

Output:
- `phase3/generated_params/<archetype>.json` — individual archetype files
- `phase3/generated_params/all_params.json` — combined file with all params,
  computed averages, and honesty scores

### Step 3: Run Simulation

```bash
python3 phase3/run_phase3.py
```

Default configuration: 25,000 hands x 20 seeds (same seeds as Phase 1 v3).
Saves to `runs_phase3.sqlite`.

Custom options:
```bash
python3 phase3/run_phase3.py --seeds 42,137,256 --hands 5000 --db custom.sqlite
```

The Dealer integrity layer runs automatically:
- Validates every action before execution
- Audits chip conservation after every hand
- Tracks VPIP/AF drift from spec targets
- Saves `dealer_audit.json` with full audit trail

### Step 4: Compare with Phase 1 (Future — Stage 6)

python3 phase3/phase3_comparison.py   # Not yet built

## Design Decisions

### Why LLM-generated parameters?

Phase 1 used hand-tuned parameter tables derived from the archetype
specification document. Phase 3 tests whether an LLM, given the same
qualitative descriptions plus quantitative targets, can independently produce
parameter tables that yield similar trust dynamics. This is a test of
**behavioral specification transfer** — can natural-language personality
descriptions be reliably converted into numerical strategies?

### Why not let the LLM play directly?

Phase 3 agents use the **same decide_action logic** as Phase 1 (probability
sampling from parameter tables). The LLM only generates the parameter tables
offline — it does not make real-time decisions during the simulation. This
design isolates the effect of parameter generation from the effect of
decision-making architecture.

### Dealer integrity layer

The Dealer provides a safety net for LLM-generated parameters that might
produce illegal or degenerate actions:
- **Pre-action validation**: Substitutes legal defaults for illegal actions
- **Post-hand audits**: Verifies chip conservation and showdown correctness
- **Anomaly detection**: Flags behavioral drift from personality spec targets

### Agent classes

Three agent classes handle the 8 archetypes:

| Class | Archetypes | Notes |
|-------|-----------|-------|
| `LLMAgent` | oracle, sentinel, firestorm, wall, phantom, mirror | Static per-round params |
| `LLMPredator` | predator | Adaptive: blends baseline → exploit when posterior > 0.60 |
| `LLMJudge` | judge | Dual-state: cooperative (default) → retaliatory (after tau=5 bluffs) |

All three inherit from Phase 1's `BaseAgent`, so they plug directly into the
existing game engine without modification.
